<objective>
## Goal
1. FolderPlus button — creates a named folder (empty) without requiring a tool to be edited first
2. Drag-to-reorder — folder sections can be reordered via drag; order persists to the org

## Architecture
- `organizations.tool_folder_order text[]` stores the canonical folder order for the org
- When the user drags folders to reorder, the array is updated via server action
- When a new folder is created via the button, its name is appended to the array
- Rendering: sort folder groups by their index in `tool_folder_order`; folders not in the array append alphabetically at the end
- The "Other" section (null folder) is always last and not draggable
</objective>

    Add two new exported functions:

    **getFolderOrder** — returns the org's current tool_folder_order:
    ```ts
    export async function getFolderOrder(): Promise<string[]> {
      const supabase = await createClient()
      const { data } = await supabase
        .from('organizations')
        .select('tool_folder_order')
        .single()
      return data?.tool_folder_order ?? []
    }
    ```

    **saveFolderOrder** — persists a new order:
    ```ts
    export async function saveFolderOrder(order: string[]): Promise<{ error?: string } | void> {
      const user = await getUser()
      if (!user) return { error: 'Not authenticated.' }
      const supabase = await createClient()
      const { error } = await supabase
        .from('organizations')
        .update({ tool_folder_order: order })
        .eq('id', (await supabase.rpc('get_current_org_id')).data)
      if (error) return { error: error.message }
      revalidatePath('/tools')
    }
    ```

    Note: `get_current_org_id()` is a SECURITY DEFINER RPC already in the DB.

    function handleDragEnd(event: DragEndEvent) {
      const { active, over } = event
