## Cross-Phase Integration: 41/42

### WARN: Campaign Calls Not in Observability List

Campaign outbound calls do not automatically appear in the Observability `/calls` list. `/api/vapi/campaigns` writes only to `campaign_contacts`, not to `calls`. The `/api/vapi/calls` route CAN receive outbound webhooks but requires separate Vapi webhook config.

**Recommendation:** Document dual-webhook requirement OR modify `/api/vapi/campaigns` to also insert into `calls` table.

### WARN: No Vapi Webhook HMAC Validation

All `/api/vapi/*` routes accept unauthenticated POSTs. Middleware correctly excludes them from cookie auth, but no `x-vapi-secret` header validation exists. Security hardening gap, not functional break.
