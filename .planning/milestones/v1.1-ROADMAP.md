# Milestone v1.1: Knowledge Base

**Status:** ✅ SHIPPED 2026-04-03
**Delivered as:** Single atomic commit (bf541bb) — all phases planned, executed, and verified in one session
**Total changes:** 18 files, +2191 / -334 lines

## Overview

Replaced the v1.0 stub knowledge base with a production-grade LangChain + Supabase vector pipeline. Files and URLs are both vectorized and searchable via pgvector, wired to each org's OpenAI integration. Per-org limits enforced server-side. UI gated on OpenAI integration presence.

## Phases

### Phase 1: Data Layer

**Focus:** Migrate DB schema to LangChain-compatible layout.

**Delivered:**
- Migration 010: renames `documents` → `knowledge_sources` (source tracking), creates new `documents` table with `content`, `metadata` (JSONB), `embedding vector(1536)`
- Drops old `document_chunks` table
- Creates `match_documents` RPC (LangChain's expected function signature)
- Updates `src/types/database.ts`

### Phase 2: File Pipeline

**Focus:** File upload → chunk → embed → store.

**Delivered:**
- `process-embeddings` Deno Edge Function upgraded to use LangChain `RecursiveCharacterTextSplitter` (1000 tokens, 100 overlap) + `OpenAIEmbeddings` (text-embedding-3-small) + `SupabaseVectorStore.fromDocuments()`
- Metadata includes `org_id`, `source_id`, `knowledge_source_id` for filtering
- Status updates: `knowledge_sources.status` set to `processing` → `ready`; `chunk_count` written

### Phase 3: URL Pipeline

**Focus:** URL → scrape → chunk → embed → store.

**Delivered:**
- `extractText()` function in Edge Function handles URLs: fetches with user-agent, strips script/style tags, cleans HTML
- Same chunking + embedding pipeline as file pipeline
- `source_type = 'url'` in `knowledge_sources`

### Phase 4: UI & Wiring

**Focus:** UI gating, limits, and semantic search wiring.

**Delivered:**
- `src/actions/knowledge.ts`: `hasOpenAiIntegration()`, `getKnowledgeSources()`, server-side limit enforcement (5 files, 5 URLs)
- `src/components/knowledge/openai-banner.tsx`: inline amber banner with link to Integrations
- `src/components/knowledge/upload-form.tsx`: disabled when OpenAI not configured; counters (X/5)
- `src/components/knowledge/document-list.tsx`: shadcn `AlertDialog` for delete (no `window.confirm()`)
- `src/lib/knowledge/query-knowledge.ts`: LangChain `SupabaseVectorStore.similaritySearch()` with `{ org_id }` filter
- `src/lib/action-engine/execute-action.ts`: `knowledge_base` case wired to `queryKnowledge()`

## Milestone Summary

**Key Decisions:**
- LangChain as the vector abstraction layer (vs raw pg calls) — keeps embedding and search code clean, community-maintained
- `metadata.org_id` for vector isolation (vs separate tables per org) — follows LangChain SupabaseVectorStore conventions
- OpenAI only for embeddings — key sourced from org integrations, no env var fallback

**Issues Resolved:**
- Migration CLI account mismatch resolved via Supabase dashboard SQL editor

**Technical Debt:**
- No URL content refresh — stale URLs require manual delete + re-add
- No failed chunk partial-recovery — if edge function errors mid-batch, source stays `processing`
- No PDF page metadata stored in chunk metadata
