Create `supabase/migrations/002_action_engine.sql` with the complete Phase 2 schema. Use the exact SQL from RESEARCH.md Pattern 5 as the source of truth, making no structural changes. Key requirements:

**ENUM types (create before tables):**
```sql
CREATE TYPE public.action_type AS ENUM (
  'create_contact',
  'get_availability',
  'create_appointment',
  'send_sms',
  'knowledge_base',
  'custom_webhook'
);

CREATE TYPE public.integration_provider AS ENUM (
  'gohighlevel',
  'twilio',
  'calcom',
  'custom_webhook'
);
```

**integrations table:**
- id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- organization_id UUID NOT NULL REFERENCES public.organizations(id) ON DELETE CASCADE
- provider public.integration_provider NOT NULL
- name TEXT NOT NULL
- encrypted_api_key TEXT NOT NULL (stores AES-256-GCM ciphertext — never plaintext)
- location_id TEXT (GHL sub-account ID — nullable)
- config JSONB NOT NULL DEFAULT '{}'
- is_active BOOLEAN NOT NULL DEFAULT true
- created_at TIMESTAMPTZ NOT NULL DEFAULT now()
- updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
- ENABLE ROW LEVEL SECURITY
- INDEX on organization_id: `CREATE INDEX idx_integrations_org_id ON public.integrations(organization_id)`
- updated_at trigger using public.update_updated_at()

**tool_configs table:**
- id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- organization_id UUID NOT NULL REFERENCES public.organizations(id) ON DELETE CASCADE
- integration_id UUID NOT NULL REFERENCES public.integrations(id) ON DELETE RESTRICT
- tool_name TEXT NOT NULL (must match Vapi tool name exactly — case-sensitive)
- action_type public.action_type NOT NULL
- config JSONB NOT NULL DEFAULT '{}'
- fallback_message TEXT NOT NULL
- is_active BOOLEAN NOT NULL DEFAULT true
- created_at TIMESTAMPTZ NOT NULL DEFAULT now()
- updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
- UNIQUE(organization_id, tool_name)
- ENABLE ROW LEVEL SECURITY
- INDEX on organization_id: `CREATE INDEX idx_tool_configs_org_id ON public.tool_configs(organization_id)`
- Composite index (hot path): `CREATE INDEX idx_tool_configs_org_tool ON public.tool_configs(organization_id, tool_name)`
- updated_at trigger

**action_logs table:**
- id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- organization_id UUID NOT NULL REFERENCES public.organizations(id) ON DELETE CASCADE
- tool_config_id UUID REFERENCES public.tool_configs(id) ON DELETE SET NULL (nullable — tool may be deleted)
- vapi_call_id TEXT NOT NULL
- tool_name TEXT NOT NULL
- status TEXT NOT NULL CHECK (status IN ('success', 'error', 'timeout'))
- execution_ms INTEGER NOT NULL
- request_payload JSONB NOT NULL DEFAULT '{}'
- response_payload JSONB NOT NULL DEFAULT '{}'
- error_detail TEXT (nullable)
- created_at TIMESTAMPTZ NOT NULL DEFAULT now()
- NO updated_at (logs are append-only)
- ENABLE ROW LEVEL SECURITY
- INDEX on organization_id: `CREATE INDEX idx_action_logs_org_id ON public.action_logs(organization_id)`
- Index on created_at DESC: `CREATE INDEX idx_action_logs_created ON public.action_logs(created_at DESC)`
- Index on tool_config_id: `CREATE INDEX idx_action_logs_tool_config ON public.action_logs(tool_config_id)`

**RLS policies for all three tables** — follow Phase 1 pattern exactly:
- integrations: SELECT, INSERT (WITH CHECK), UPDATE (USING + WITH CHECK), DELETE
- tool_configs: SELECT, INSERT (WITH CHECK), UPDATE (USING + WITH CHECK), DELETE
- action_logs: SELECT, INSERT (WITH CHECK) — no UPDATE or DELETE (logs are immutable)

All policies use: `(SELECT public.get_current_org_id()) = organization_id`

Note: The webhook route uses service-role key (bypasses RLS by design — no user JWT in Vapi requests). RLS policies only apply to admin UI server actions.
